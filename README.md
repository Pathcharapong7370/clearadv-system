# ระบบเคลียร์เงินทดรองจ่าย (FM-AC-05 / FM-AC-06)

เว็บแอปสำหรับกรอก บันทึก และค้นหา:

- **FM-AC-05** ใบสรุปเคลียร์เงินทดรองจ่าย (มีบิล/ไม่มีบิล แยกตามประเภทค่าใช้จ่าย)
- **FM-AC-06** ใบสรุปค่าใช้จ่าย (ไม่มีเอกสาร)

หน้าเว็บเป็น HTML/JS ล้วน (ไม่ต้อง build ไม่ต้องใช้ Node) เก็บข้อมูลจริงไว้ใน **Supabase**
(ฐานข้อมูล Postgres + ระบบล็อกอิน) และโฮสต์เว็บฟรีผ่าน **GitHub Pages**

ไม่ต้องมีความรู้ด้านโปรแกรมมิ่งมาก่อนก็ทำตามได้ ใช้เวลาประมาณ 20-30 นาที

---

## ภาพรวมขั้นตอนทั้งหมด

1. สร้างโปรเจกต์ Supabase (ฟรี) + สร้างตารางฐานข้อมูล
2. สร้างผู้ใช้สำหรับล็อกอินเข้าเว็บ
3. คัดลอกกุญแจ (API Key) ของ Supabase มาใส่ในไฟล์ `assets/config.js`
4. อัปโหลดโค้ดทั้งหมดขึ้น GitHub
5. เปิดใช้งาน GitHub Pages เพื่อให้ได้ลิงก์เว็บไซต์
6. ทดสอบเข้าใช้งานจริง

---

## ขั้นตอนที่ 1: สร้างโปรเจกต์ Supabase

1. เข้าเว็บ https://supabase.com แล้วกด **Start your project** เพื่อสมัคร/ล็อกอิน (สมัครด้วย GitHub ได้เลย)
2. กด **New project**
   - **Name**: ตั้งชื่อ เช่น `clearadv-system`
   - **Database Password**: ตั้งรหัสผ่านที่คาดเดายาก แล้ว **จดเก็บไว้ให้ดี** (ใช้กรณีต้องต่อฐานข้อมูลโดยตรงในอนาคต)
   - **Region**: เลือก `Southeast Asia (Singapore)` เพื่อความเร็วในการเชื่อมต่อจากไทย
   - กด **Create new project** แล้วรอประมาณ 1-2 นาทีจนโปรเจกต์พร้อมใช้งาน

## ขั้นตอนที่ 2: สร้างตารางฐานข้อมูลด้วยไฟล์ SQL

1. ในเมนูซ้ายของ Supabase Dashboard กด **SQL Editor**
2. กด **New query**
3. เปิดไฟล์ `supabase/schema.sql` ที่แนบมาด้วยตัวแก้ไขข้อความ (Notepad ก็ได้) คัดลอกทั้งหมด
4. วางลงในช่อง SQL Editor ของ Supabase
5. กดปุ่ม **Run** (หรือ Ctrl+Enter)
6. ควรเห็นข้อความ `Success. No rows returned` แปลว่าเรียบร้อย

ไฟล์นี้จะสร้างตารางทั้งหมด 4 ตาราง (`adv_clear_forms`, `adv_clear_items`, `exp_noreceipt_forms`,
`exp_noreceipt_items`) พร้อมเปิดระบบความปลอดภัยระดับแถว (Row Level Security) ให้อัตโนมัติ
โดยกำหนดว่า **เฉพาะผู้ที่ล็อกอินแล้วเท่านั้นจึงจะอ่าน/บันทึกข้อมูลได้**

> ตรวจสอบผลลัพธ์ได้ที่เมนู **Table Editor** ทางซ้าย ควรเห็นตารางทั้ง 4 ตารางขึ้นมาแล้ว

## ขั้นตอนที่ 3: สร้างผู้ใช้สำหรับล็อกอินเข้าเว็บ

เว็บนี้ล็อกอินด้วยอีเมล + รหัสผ่านผ่านระบบ Supabase Auth (ไม่มีหน้าสมัครสมาชิกเอง เพื่อความปลอดภัย
ผู้ดูแลระบบเป็นคนสร้างบัญชีให้พนักงานเอง)

1. ในเมนูซ้าย กด **Authentication** > **Users**
2. กด **Add user** > **Create new user**
3. กรอกอีเมลและรหัสผ่านของพนักงานที่จะใช้งาน (ทำซ้ำได้หลายคน)
4. ที่ช่อง **Auto Confirm User** ให้ติ๊กถูก (เพื่อไม่ต้องยืนยันอีเมล ใช้ล็อกอินได้ทันที)
5. กด **Create user**

ทำซ้ำขั้นตอนนี้สำหรับพนักงาน/ผู้ที่ต้องใช้งานเว็บทุกคน

## ขั้นตอนที่ 4: นำกุญแจ Supabase มาใส่ในโค้ด

1. ในเมนูซ้าย กด **Project Settings** (รูปเฟือง) > **API**
2. คัดลอกค่า 2 ค่านี้:
   - **Project URL** (หน้าตาเช่น `https://abcdefghijk.supabase.co`)
   - **Project API keys > anon public** (ข้อความยาว ๆ ขึ้นต้นด้วย `eyJ...`)
3. เปิดไฟล์ `assets/config.js` ในโค้ดที่แนบมา แก้ไข 2 บรรทัดนี้:

   ```js
   window.SUPABASE_URL = "https://abcdefghijk.supabase.co";
   window.SUPABASE_ANON_KEY = "eyJ...ใส่ค่าที่คัดลอกมา...";
   ```

4. บันทึกไฟล์

> **หมายเหตุด้านความปลอดภัย**: `anon public key` ออกแบบมาให้ฝังในโค้ดฝั่งเว็บ (frontend) ได้อย่างปลอดภัย
> เพราะการเข้าถึงข้อมูลจริงถูกบังคับด้วย Row Level Security ที่ตั้งไว้ในขั้นตอนที่ 2 แล้ว
> (ต้องล็อกอินก่อนเท่านั้นถึงจะอ่าน/เขียนข้อมูลได้) **ห้ามใช้ค่า `service_role key` ในเว็บฝั่งหน้าบ้านเด็ดขาด**

## ขั้นตอนที่ 5: อัปโหลดโค้ดขึ้น GitHub

### 5.1 สร้าง repository ใหม่
1. เข้า https://github.com แล้วล็อกอิน (สมัครฟรีได้ที่ https://github.com/signup หากยังไม่มีบัญชี)
2. กดเครื่องหมาย **+** มุมขวาบน > **New repository**
3. ตั้งชื่อ เช่น `clearadv-system`
4. เลือก **Public** (จำเป็นสำหรับ GitHub Pages แบบฟรี) หรือ **Private** ก็ได้หากมีแผน GitHub Pro/Team
5. **ไม่ต้อง** ติ๊ก "Add a README file" (เรามีไฟล์ README มาให้แล้ว)
6. กด **Create repository**

### 5.2 อัปโหลดไฟล์ (เลือกวิธีใดวิธีหนึ่ง)

**วิธี A: อัปโหลดผ่านหน้าเว็บ (ง่ายที่สุด ไม่ต้องติดตั้งอะไร)**

1. ในหน้า repository ที่สร้างใหม่ กด **uploading an existing file**
2. ลากไฟล์/โฟลเดอร์ทั้งหมดนี้ไปวาง (`index.html`, โฟลเดอร์ `assets/`, โฟลเดอร์ `supabase/`, `README.md`)
   - ให้แน่ใจว่าโครงสร้างโฟลเดอร์ยังเหมือนเดิม เช่น `assets/app.js`, `assets/style.css`, `assets/config.js`
3. เลื่อนลงล่าง ใส่ข้อความ commit เช่น `เพิ่มระบบเคลียร์เงินทดรองจ่าย`
4. กด **Commit changes**

**วิธี B: ใช้ Git ผ่านคอมพิวเตอร์ (สำหรับผู้ที่คุ้นเคยกับ command line)**

```bash
cd path/to/clearadv-app
git init
git add .
git commit -m "เพิ่มระบบเคลียร์เงินทดรองจ่าย"
git branch -M main
git remote add origin https://github.com/<ชื่อผู้ใช้ของคุณ>/clearadv-system.git
git push -u origin main
```

## ขั้นตอนที่ 6: เปิดใช้งาน GitHub Pages

1. ในหน้า repository กด **Settings** (แท็บบนสุด)
2. เมนูซ้าย เลื่อนลงไปกด **Pages**
3. หัวข้อ **Build and deployment > Source** เลือก **Deploy from a branch**
4. หัวข้อ **Branch** เลือก `main` และโฟลเดอร์ `/ (root)` แล้วกด **Save**
5. รอประมาณ 1-2 นาที รีเฟรชหน้า จะเห็นข้อความสีเขียวพร้อมลิงก์ เช่น
   `Your site is live at https://<ชื่อผู้ใช้>.github.io/clearadv-system/`
6. คลิกลิงก์นั้นเพื่อเปิดเว็บ

## ขั้นตอนที่ 7: ทดสอบใช้งาน

1. เปิดลิงก์เว็บที่ได้จากขั้นตอนที่ 6
2. หน้าล็อกอินจะขึ้นมา ให้กรอกอีเมล/รหัสผ่านที่สร้างไว้ในขั้นตอนที่ 3
3. เมื่อเข้าสู่ระบบสำเร็จ จะเห็นแท็บ **FM-AC-05** และ **FM-AC-06**
4. ลองกรอกข้อมูล กดปุ่ม **บันทึกลง Supabase** แล้วตรวจสอบว่าข้อมูลขึ้นในตาราง **Table Editor** ของ Supabase จริง
5. ปุ่ม **พิมพ์ / PDF** ใช้สำหรับพิมพ์เอกสารหรือบันทึกเป็น PDF ผ่านหน้าต่างพิมพ์ของเบราว์เซอร์

## โครงสร้างไฟล์

```
clearadv-app/
├── index.html              หน้าเว็บหลัก
├── assets/
│   ├── config.js            ใส่ค่า Supabase URL/Key ที่นี่ (แก้ไฟล์เดียวพอ)
│   ├── style.css            ดีไซน์ของเว็บ
│   └── app.js                ลอจิกทั้งหมด: ล็อกอิน, ฟอร์ม, คำนวณยอด, เชื่อม Supabase
├── supabase/
│   └── schema.sql            สคริปต์สร้างตารางฐานข้อมูล + สิทธิ์การเข้าถึง
└── README.md                 ไฟล์นี้
```

## การเพิ่ม/ลบผู้ใช้งานภายหลัง

ทำได้ที่ Supabase Dashboard > **Authentication > Users** ตลอดเวลา โดยไม่ต้องแก้โค้ดหรือ deploy ใหม่
- เพิ่มผู้ใช้: **Add user > Create new user** (ติ๊ก Auto Confirm User)
- ลบผู้ใช้: กดสามจุดท้ายแถวผู้ใช้ > **Delete user**
- ผู้ใช้ลืมรหัสผ่าน: กดสามจุดท้ายแถว > **Send password recovery** หรือกำหนดรหัสผ่านใหม่ให้โดยตรง

## การแก้ไขปัญหาที่พบบ่อย

| อาการ | สาเหตุที่เป็นไปได้ / วิธีแก้ |
|---|---|
| หน้าเว็บขึ้นค้างที่ "กำลังโหลด..." | เปิด Developer Console (กด F12) ดู error ส่วนใหญ่มาจาก `SUPABASE_URL`/`SUPABASE_ANON_KEY` ใน `config.js` ยังไม่ได้แก้ หรือพิมพ์ผิด |
| ล็อกอินแล้วขึ้น "Invalid login credentials" | ตรวจสอบว่าสร้างผู้ใช้ใน Supabase แล้วจริง และติ๊ก Auto Confirm User ไว้ตอนสร้าง |
| กดบันทึกแล้วขึ้น error เกี่ยวกับ RLS/permission | ตรวจสอบว่ารัน `supabase/schema.sql` ครบทุกคำสั่งแล้ว โดยเฉพาะส่วน `enable row level security` และ `create policy` ท้ายไฟล์ |
| เปิดลิงก์ GitHub Pages แล้วขึ้น 404 | รอเพิ่มอีกสักครู่ (การ deploy ครั้งแรกใช้เวลา 1-2 นาที) หรือตรวจสอบว่าเลือก Branch/โฟลเดอร์ถูกต้องในขั้นตอนที่ 6 |
| แก้โค้ดแล้วเว็บไม่เปลี่ยน | อัปโหลดไฟล์ใหม่ทับของเดิมใน GitHub (commit ใหม่) แล้วรอ GitHub Pages deploy ซ้ำ ~1 นาที หรือลอง Hard Refresh (Ctrl+Shift+R) |
| ตัวเลขยอดรวมไม่อัปเดต | ตรวจสอบว่าไม่ได้แก้ไขโครงสร้าง HTML ของ `app.js` โดยไม่ตั้งใจ ลองโหลดหน้าใหม่ (F5) |

## แนวทางต่อยอดในอนาคต (ถ้าต้องการ)

- เพิ่มการอัปโหลดไฟล์ Slip การโอนเงินจริง โดยใช้ **Supabase Storage** (สร้าง bucket แล้วอัปโหลดผ่าน `sb.storage`)
- เพิ่มระบบแจ้งเตือนอีเมลเมื่อมีการอนุมัติ ผ่าน **Supabase Edge Functions**
- เพิ่มสิทธิ์แยกตามบทบาท (ผู้ขอเบิก/ผู้อนุมัติ/บัญชี) ด้วยตาราง `profiles` และเงื่อนไขใน RLS Policy
- Export ข้อมูลเป็น Excel โดยดึงข้อมูลจาก Supabase มาสร้างไฟล์ .xlsx ฝั่ง client (เช่นไลบรารี SheetJS)
